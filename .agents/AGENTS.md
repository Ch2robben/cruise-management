# 长航集团游轮管理系统 — B 端交互风格规范

## 核心原则：B 端 ≠ C 端

本系统是面向内部运营人员的管理后台（B 端），不是面向消费者的展示页面（C 端）。
所有新功能、改版、弹窗设计时，优先考虑「效率」和「信息密度」，而不是「视觉冲击力」和「动效体验」。
本项目为demo演示
---

## B 端 vs C 端 对照

| 维度 | B 端（要做） | C 端（要避免） |
|------|-------------|---------------|
| 数据展示 | 表格（DataTable），列头清晰，支持排序 | 卡片流、大图瀑布流 |
| 导航 | 层级清晰的侧边栏 + 面包屑 | 底部 Tab 栏、横向滑动导航 |
| 交互流程 | 弹窗表单 + Tab 键批量录入 | 步骤引导动画、全屏沉浸式流程 |
| 颜色 | 中性灰为底色，蓝色高亮主操作，红色警示危险 | 渐变大图、高饱和配色、玻璃拟态 |
| 动效 | 无或极短（≤150ms 淡入），不分散注意力 | 复杂过渡动画、弹跳效果、视差滚动 |
| 布局 | 固定列宽表格、左侧标签+右侧内容、垂直滚动 | 横向滚动时间轴、卡片横滑、全屏轮播 |
| 信息密度 | 高密度，多字段并排，折叠分组 | 单内容居中大字，大量留白 |
| 操作反馈 | Toast 轻提示、行内状态更新 | 全屏成功页、庆祝动画 |

## 具体约束

### 布局
- 列表型数据用 `DataTable`，弹窗内多行数据用 `<table>`，不用卡片。
- 弹窗内有多个逻辑分区时，用可折叠 `<section>`（Accordion），默认展开。
- 弹窗宽度：普通表单 `max-w-2xl`，复杂多列 `max-w-4xl`，超出改用右侧抽屉。
- 纵向滚动优先；若横向节点很多（如时间轴），改为纵向分组表格。

### 配色
- 背景：白色 + 浅灰（`bg-gray-50`）
- 主操作：`bg-blue-600`；危险：`bg-red-600`；次要：白底灰边框
- 不使用高饱和渐变色块、glassmorphism 或大色块背景作为容器

### 录入体验
- 表格内 `input` 保证 Tab 键顺序可用，支持键盘批量录入
- 只读字段：`bg-gray-50` 无边框，视觉上与可编辑字段区分
- 必填字段 label 后跟红色 `*`，不用 placeholder 替代 label

### 字段规范
- 物理距离（km）：上下水方向一致，只展示一列
- 航行时间：上水/下水时间不同，各展示一列（分钟，纯数字录入）
- 时间格式化输出统一用 `formatDurationMinutes(min)`

### 操作规范
- 主操作（保存）在右下角；取消在左或次要位置
- 破坏性操作（删除、重置）须二次确认（`ConfirmDialog`）
- 状态展示统一用 `StatusBadge` 组件
