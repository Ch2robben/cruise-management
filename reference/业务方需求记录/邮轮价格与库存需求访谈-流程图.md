# 邮轮价格与库存流程图

> 基于：`邮轮价格与库存需求访谈-meeting-notes.md`、`邮轮价格与库存需求访谈-产品设计思路.md`
> 用途：辅助原型设计、PRD 编写和研发对齐。

## 1. 整体业务主流程

```mermaid
flowchart TD
  A["配置基础数据"] --> B["配置航线/航段/航次"]
  B --> C["配置房型与入住规则"]
  C --> D["配置航段基准价"]
  D --> E["配置价格政策"]
  E --> F["配置政策库存与优先级"]
  F --> G["经销商/经营人员售卖"]
  G --> H["订单试算"]
  H --> I{"价格与库存是否满足"}
  I -- "满足" --> J["生成订单"]
  I -- "不满足" --> K["提示不可预订或回退政策"]
  K --> H
  J --> L["支付/充值额度扣减"]
  L --> M["保存订单计价快照"]
  M --> N["财务对账/提成报表"]
```

## 2. 订单价格计算流程

```mermaid
flowchart TD
  A["选择航次"] --> B["选择航段"]
  B --> C["选择房型"]
  C --> D["录入游客信息"]
  D --> E["识别游客类型"]
  E --> F["成人/儿童/婴儿/外宾"]
  F --> G["按入住优先级排序"]
  G --> H["判断第几人入住"]
  H --> I["判断占床/加床/不占床"]
  I --> J{"是否存在入住价格规则"}
  J -- "不存在" --> K["禁止预订并提示缺少价格规则"]
  J -- "存在" --> L["读取航段房型基准价"]
  L --> M["套用入住序位价格公式"]
  M --> N["得到每位游客基础价格"]
  N --> O["进入价格政策匹配"]
```

## 3. 价格政策命中与回退流程

```mermaid
flowchart TD
  A["输入订单上下文"] --> B["航线/航段/航次"]
  A --> C["经销商/经营人员"]
  A --> D["游客地区/国家"]
  A --> E["房型/入住类型"]

  B --> F["匹配可用价格政策"]
  C --> F
  D --> F
  E --> F

  F --> G{"是否命中政策"}
  G -- "未命中" --> H["使用基础价格"]
  G -- "命中" --> I["按政策优先级排序"]
  I --> J["取最高优先级政策"]
  J --> K{"政策库存是否可用"}
  K -- "可用" --> L["应用政策价格"]
  K -- "不可用" --> M{"是否存在下一优先级政策"}
  M -- "存在" --> J
  M -- "不存在" --> H

  L --> N["扣减政策库存"]
  H --> O["生成最终价格"]
  N --> O
  O --> P["记录命中政策与计算过程"]
```

## 4. 库存判断流程

```mermaid
flowchart TD
  A["订单试算/下单"] --> B["判断房间库存"]
  B --> C{"房间是否可售"}
  C -- "否" --> D["不可预订"]
  C -- "是" --> E["判断床位库存"]
  E --> F{"占床人数是否超出可用床位"}
  F -- "是" --> D
  F -- "否" --> G["判断核载人数"]
  G --> H{"总人数是否超出核载上限"}
  H -- "是" --> D
  H -- "否" --> I["判断政策库存"]
  I --> J{"优惠名额是否可用"}
  J -- "可用" --> K["按优惠价预订"]
  J -- "不可用" --> L["回退下一政策或基础价"]
  L --> M{"回退后价格是否可用"}
  M -- "否" --> D
  M -- "是" --> N["按回退价格预订"]
  K --> O["锁定库存"]
  N --> O
```

## 5. 房型入住计价流程

```mermaid
flowchart TD
  A["录入同一房间游客"] --> B["识别游客类型"]
  B --> C["成人"]
  B --> D["儿童"]
  B --> E["婴儿"]

  C --> F["按优先级排序"]
  D --> F
  E --> F

  F --> G["成人优先占床"]
  G --> H["儿童其次占床/加床/不占床"]
  H --> I["婴儿最后判断"]
  I --> J["生成入住序位"]
  J --> K["第1人/第2人/第3人/第4人"]
  K --> L{"房型是否支持该序位"}
  L -- "不支持" --> M["禁止预订"]
  L -- "支持" --> N{"是否配置价格公式"}
  N -- "未配置" --> M
  N -- "已配置" --> O["按公式计算价格"]
  O --> P["写入游客价格明细"]
```

## 6. 手动充值与水单风控流程

```mermaid
flowchart TD
  A["经销商/经营人员发起手动提单"] --> B["填写转账信息"]
  B --> C["上传或补充水单信息"]
  C --> D["提交财务审核"]
  D --> E{"财务是否审核通过"}
  E -- "驳回" --> F["退回修改或关闭"]
  E -- "通过" --> G["增加经销商可用额度"]
  G --> H{"水单是否已匹配"}
  H -- "已匹配" --> I["结单完成"]
  H -- "未匹配" --> J["进入待匹配状态"]
  J --> K{"是否超过提醒期限"}
  K -- "否" --> J
  K -- "是" --> L["提醒经销商"]
  L --> M["提醒经营人员"]
  M --> N["提醒财务"]
  N --> O{"是否补齐并匹配成功"}
  O -- "是" --> I
  O -- "否" --> P["人工结单"]
  P --> Q["填写未匹配原因"]
  Q --> I
```

## 7. 组合产品结算流程

```mermaid
flowchart TD
  A["用户购买邮轮产品"] --> B{"是否选择搭售产品"}
  B -- "否" --> C["生成普通邮轮订单"]
  B -- "是" --> D["生成组合订单"]
  D --> E["拆分主产品与搭售产品金额"]
  E --> F{"是否支持线上分账"}
  F -- "暂不支持" --> G["统一收款到主账户"]
  G --> H["财务定期线下结算"]
  F -- "支持" --> I["按分账规则线上分账"]
  I --> J["生成分账记录"]
  H --> K["订单结算完成"]
  J --> K
  C --> K
```

## 8. 配置后台页面流转

```mermaid
flowchart LR
  A["基础数据"] --> B["船舶/航线/航段"]
  B --> C["房型/入住类型"]
  C --> D["入住价格规则"]
  D --> E["航段基准价"]
  E --> F["价格政策"]
  F --> G["政策优先级"]
  G --> H["政策库存"]
  H --> I["订单试算器"]

  J -- "通过" --> K["进入原型/PRD定稿"]
  J -- "不通过" --> D
```

## 9. 推荐原型验证路径

flowchart TD

  B --> C["输入经销商"]

  D --> E["展示入住序位"]
