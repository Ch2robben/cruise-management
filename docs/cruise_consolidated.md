# 第一章 业务概述

## 1.1 游轮业务定义

长江游轮业务的核心特征是**跨城航行**和**船上住宿**，区别于城市游船（同城观光、无住宿）。主要业务场景包括：

- 游客购买船票+住宿+岸上游的综合旅游产品
- 船上二次消费（餐饮、购物、娱乐、岸上自费项目等）
- 通过官网、小程序、OTA、同业分销、线下经销商等多渠道销售
- 以"航次"为核心运营单位，按单船单航次进行收入、成本、利润核算

## 1.2 涉及系统全景

当前长江游轮业务涉及以下系统：

| 系统 | 定位 | 当前状态 |
|------|------|----------|
| 游轮运营管理系统（游管系统） | 游轮运营核心系统 | 建设中（即本项目） |
| ICS 票务系统 | 船票销售、舱位管理 | 运行中 |
| 星瀚系统 | 财务核算（应收/应付/总账） | 运行中 |
| 壹招报系统 | 费用报销 | 运行中 |
| 船舶管理系统 | 船舶技术管理 | 运行中 |
| 船员系统 | 船员人事管理 | 运行中 |
| OA 系统 | 审批流程 | 运行中 |
| 小程序/官网 | C端销售入口 | 运行中 |

一体化平台的建设目标是：以游管系统为核心，整合上述系统，实现业务与财务（业财）一体化。

# 第二章 销售与渠道管理

## 2.1 多渠道销售能力

### 2.1.1 操作界面

- 手机端轻量化设计，参照 12306 交互逻辑，核心操作 ≤3 步，适配老年用户
- 电脑端保留全功能模块，侧重批量操作、复杂审批、数据统计
- 多终端适配：手机、平板、POS 端，与电脑端功能互补
- 支持 7 种语言：简体中文、繁体中文、英文、法文、日文、韩文、俄文

### 2.1.2 预订功能

- 支持对同业（B端）、对渠道（B端）、对散客（C端）的基本预订功能
- B端需分别定义外部人员和内部人员的操作流程与权限边界
- C端需明确内部人员与外部人员的操作路径与交互逻辑

### 2.1.3 销售工具

- 专题海报、优惠券、补差助手、直连景区门票系统、在线客服、消息通知、短链接生成
- 支持与现有票务系统、游管系统、POS 机等直连对接
- 2期扩展：佣金助手、直播、社群营销

## 2.2 舱位管理

- 实现**床位库存**与**额定载客数量**的双库存管理
- 实时展示各游轮额定载客量、可用状态、占用情况
- 支持多级库存预警
- 系统内置 OA 审批流程：审批通过后自动执行订单修改操作，并记录审批日志

## 2.3 价格管理

- 支持零售公布价、线上销售价、签约结算价、区域结算价、价格策略的配置
- 实现 OTA 服务费统一叠加投放
- 支持游轮产品价格按多维度（月、星期、日）批量上架与调整
- 价格可通过 API 直连数据或手工编辑上架并行
- 2期：支持多币种标价与实时汇率换算

## 2.4 客户管理

- 建立完整客户档案：身份证、手机号等识别细化至城市级别
- 支持客户分层与精准客源分析
- 2期：客户画像增加国籍分析、购买人与乘船人关系分析

## 2.5 会员管理

- 会员体系分前、中、后三台：后台在 ICS 系统，前端在小程序和官网
- 需明确会员界定条件、来源渠道
- 支持：会员数据查询、详情查看、标签（添加/编辑/删除）、会员卡、会员等级、积分任务管理、积分商城

# 第四章 运营管理

## 4.1 航次排班与船舶负载

- 航线信息、航次排班、舱位类型及数量、在船服务、船舶负载等数据集中管理
- 对接渠道合作数据：官网、小程序、OTA、线下经销商、口岸终端的运营及交易数据

## 4.2 销售交易数据

- 一销数据（船票）、二销零售（船上消费）、订单详情、支付退改记录、库存变动
- 所有消费数据统一口径对齐到消费人员

## 4.3 自动分团→分桌→分车

酒店运营组提出的自动化流程需求：

**核心原则**：以团为单位不打散原有团队；容量优先，不超桌、不超员。支持一键自动分配+人工微调，可直接调用已分团数据。

**自动分桌（餐厅用餐）**：
- 基础配置：维护餐厅、场次、桌号、座位数、可用状态
- 分配逻辑：按团整桌分配，大团拆分连续桌，小团可拼桌
- 输出：分桌清单、团队用餐单，可打印/导出

**自动分车（岸上游/接驳）**：
- 基础配置：维护车牌号、核载人数、司机、行程信息
- 分配逻辑：整团优先分配，大团分多车，小团合并拼车
- 输出：派车单、车辆分配表、导游清单，可打印/导出

**系统要求**：
- 与现有分团模块数据实时同步
- 支持 600 人内快速分配
- 保留操作日志，支持权限管理
- 生成标准打印、导出报表

# 第六章 餐饮管理

## 6.1 菜单技术规格书管理

- 技术规格书的餐料单品与采购单品对应和联动
- 结合餐料消耗数据与游客满意度反馈，优化配料标准与餐标标准
- 推动"制定-执行-评估-优化"的动态闭环管理
- 实现满意度、标准（技术规格标准、餐标标准）、成本三者之间的合理平衡

## 6.2 餐物料采购与库存

- 自动生成餐料采购计划
- 餐料领用和库存数据与技术规格书单品关联
- 反映餐料使用与菜单出品的关系
- 便于加强航次预算执行统计与成本管控分析
- 库存预警功能

## 6.3 厨余管理

- 餐厅和厨房每餐后对厨余使用电子秤分菜品称重
- 称重数据系统自动记录
- 系统根据技术规格书的餐料组成和采购价格自动计算每一类餐料的厨余重量及价值
- 自动生成菜品和餐料使用分析图，反映菜品、厨余、餐料成本与客源地、载客人数等因素的关系

## 6.4 供应商管理

- 开发短信功能，向供应商发送预计到港时间，提醒及时备货
- 供应商可在线查询公司订单状态、历史配送记录及评价结果
- 增强供需双方协同效率，提升供应链透明度与协作水平

# 第七章 财务管理（业财一体化）

## 7.1 销售与收款管理

### 7.1.1 收款自动化对账

**长江游轮**：

- 平台需支持与 OTA、同业分销等渠道的系统对接，每日自动获取渠道流水及账单
- 打通银企直连，实时获取银行收款流水
- 系统自动勾稽资金流（银行流水）与信息流（订单），匹配成功标记为"已对账"，失败生成差异报告（含金额差异、时间差异、无对应订单等类型），推送至财务人员处理
- 自动充值/第三方收款平台收款：对接第三方收款平台，自动获取银行流水，银行流水自动关联充值金额，形成对账单，完成确认后推送收款单至星瀚。推送应收单后，实现同一客户的应收单与收款单的自动结算，并推送结算单至星瀚
- 手动充值：对公转账对接星瀚，自动获取银行流水，完成充值与收款关联，确认后推送收款单至星瀚
- OTA 第三方收款平台暂收款、银行未到款的情况，需定制特定方案推送对应单据至星瀚
- 二消业务收款平台可自动按照规则将银行流水拆分认领到航次、船舶

### 7.1.2 自动化收入确认

- 系统内置收入确认规则（航次结束、订单状态变为"已完成"或服务核销后自动触发），自动生成收入确认应收工单，推送至星瀚系统
- 支持与税务管理系统对接，根据确认收入信息自动生成待开票数据，实现一键开票
- 精细化开票策略：订单金额可拆分、可分别开具多个对方单位、可部分金额分别开票、可多个订单批量勾选开票、可显示已开票与未开票金额、可红冲及重开；代金券业务开票票面需显示折扣行；销售返利等特定业务可红冲前期已开具发票
- 系统提供散客自助开票 H5 平台，支持扫码/链接访问，多维度订单检索，智能开票向导
- 统一收入确认标准，确保业务系统与财务系统采用同一套收入确认逻辑

### 7.1.3 财务单据处理

- 针对游船、游轮、增值服务订单数据，系统自动生成应收单
- 支持按经销商/旅行社/企业客户批量处理
- 销售返利结算单支持自动生成与审批，记录完整审批日志
- 所有财务单据支持批量导出与推送至星瀚，实现业财一体化
- 单据推送星瀚后，平台自动更新获取星瀚单据状态
- 被退回单据可再进行修改并再次推送
- 已推送成功的单据可红冲重推

### 7.1.4 客户退款与预收款管理

- 预收账款管理：系统支持预收款项（合同负债）的自动认领——客户付款后资金暂挂为合同负债，与订单状态强关联；订单完成后自动结算核销
- 客户退款：建立标准化在线退款审批流程。银行已收到款的场景——平台发起退款，审批通过后自动充值负数并推送付款单至星瀚；款项暂存第三方、银行未收到款的场景——退款由平台发起，审批通过后款项退回至账户，推送对应冲回单至星瀚

### 7.1.5 现金收款管理

- 航次结束后，船上会计将现金存入银行
- 系统自动抓取银行存款流水，与航次应收（如船上二消订单）进行勾稽核销
- 系统需支持现金收款与订单的关联，确保每笔现金收款能对应到具体航次、产品、客户（散客可建立匿名客户档案）

### 7.1.6 销售返利业务

- 根据具体规则每月自动生成暂估单
- 年底自动红冲，并发起销售返利实际应付单至星瀚

### 7.1.7 渠道管理

- 平台客户名称/编码自动获取集团主数据平台客户名称/客户编码/供应商编码
- 实现数据自动对接

## 7.2 采购与成本管理

### 7.2.1 成本端业财流程整合

- 当前成本端数据和业财对接流程分散在游轮运营管理系统、船舶管理系统、船员系统、壹招报系统、星瀚应付模块等
- 一体化平台应整合成本端业财流程：原业务系统仅保留业务功能，涉及业财的流程全部整合至一体化平台
- 在一体化平台中设置成本模块（除游管系统已承载的成本项目），与法务系统、招采平台集成
- 实现从采购申请、合同、付款的全流程线上化，嵌入成本与采购预算的实时管控
- 与法务系统交互：将已生效的合同信息同步至一体化平台；系统可根据客户或供应商自动匹配合同，支持一个客户/供应商对应多个合同的手工干预

### 7.2.2 成本分摊引擎

- 支持配置多种成本分摊规则：按舱位占比、按消费额、按人数、按航次等
- 公共成本（如基地成本）在录入时指定分摊范围（如按艘船、多个航次）
- 系统根据预设规则自动计算各成本中心/产品的应分摊金额，并生成分摊凭证
- 分摊结果可查询、追溯，支持手工调整（需审批）

### 7.2.3 采购付款与账务处理

- 根据发票校验结果，与采购订单、入库单匹配后生成应付工单
- 推送至星瀚系统审批并完成付款及账务处理

# 第八章 数据应用与分析

## 8.1 数据统一管理

### 8.1.1 多维度运营数据

**游轮运营数据**：航线信息、航次排班、舱位类型及数量、在船服务、船舶负载等

**渠道合作数据**：官网、小程序、OTA、线下经销商、口岸终端的运营及交易数据

### 8.1.2 全维度用户数据

- 销售交易数据：一销、二销零售、订单详情、支付退改记录、库存变动，所有消费数据需统一口径对齐到消费人员
- 用户会员数据：个人基础信息、证件信息、积分权益
- 评价反馈数据：全渠道用户评价、服务意见、品牌舆情反馈
- 基于消费数据分析游客人群出行记录、消费习惯、行为偏好

### 8.1.3 行业数据采集

- 从外部系统获取长江游轮行业大盘运营数据
- 重点采集细分市场中船票价格 3000 元以上船舶的运营数据、产品定价、行程安排、用户评价
- 为企业竞品分析、产品定价提供数据参考

### 8.1.4 数据质量管控

- 全平台采用同一套数据标准，统一数据口径、计算规则和统计方式
- 系统自动完成数据去重、纠错、敏感信息脱敏
- 实时监控数据质量，对异常数据自动标记预警
- 所有数据的来源、加工、修改记录全程可查，满足审计溯源需求

### 8.1.5 用户账号统一

- 一个用户对应一个唯一账号
- 整合小程序、OTA、船上、线下全渠道会员信息
- 形成完整用户档案，涵盖订单、消费、积分、权益
- 会员数据全平台实时同步

### 8.1.6 数据查询周期

- 放宽后台业务订单数据、资金流水记录等关键数据的查询时间跨度
- 支持按任意时间区间筛选
- 支持查看 3 年以上历史数据

## 8.2 业财核心维度与指标体系

### 8.2.1 核心维度

| 维度 | 说明 |
|------|------|
| 组织架构 | 公司、部门、业务点 |
| 业务板块 | 游船、门票、酒店、二消等（游轮部分清晰区分） |
| 合同 | 采购合同、销售合同 |
| 订单 | 销售订单、采购订单 |
| 产品 | 航线、门票类型、商品 |
| 航次 | 船名、航次号、起止时间 |
| 船舶 | 船名、吨位、舱位信息 |
| 人员 | 员工、船员 |
| 客商 | 客户、供应商 |

上述维度需在系统建设初期嵌入数据模型，确保船上每一笔收入或支出均能关联至具体航次、船舶、产品、客户。

### 8.2.2 游轮专属指标体系

| 指标 | 计算口径 |
|------|----------|
| 单船单航次收入、成本、毛利、经常性利润 | 按艘船 × 航次维度 |
| 人均对客成本 | 人均天餐料成本、人均天客用品成本等 |
| 航次航行成本 | 单航次航行直接成本 |
| 应收账款周转天数 | 按客户/渠道维度 |
| 预算执行率 | 按船舶/项目维度 |
| 存货周转率 | 按存货类别/仓库维度 |

## 8.3 财务专属分析视图

在经营分析仪表盘中增加"财务核心指标"视图：

- **单船单航次损益表**：按单船单航次展示收入、成本、毛利、利润，支持穿透至明细订单/成本
- **预算执行分析**：按船舶/项目展示预算使用进度、剩余额度、预警提示
- **应收账款账龄分析**：按客户、渠道展示应收账款账龄结构及逾期情况
- **存货周转分析**：按存货类别、仓库展示周转率及库龄
- **成本结构分析**：按成本类型、分摊对象展示成本构成及变动趋势
- 支持自定义报表，用户可拖拽维度、指标生成个性化报表，支持导出

## 8.4 数据可视化与报表

- 以折线图、柱状图、数据看板等直观形式展示数据
- 工作人员可自主搭建专属查看页面
- 自动生成日、周、月、季、年度标准报表，涵盖运营、财务、会员、评价等核心板块
- 支持自定义报表制作，一键导出 Excel、PDF 格式

## 8.5 营销数据分析

- **客户画像**：性别占比、年龄分析、前往不同游轮分析占比、游客行为分析、客源地分析
- **营销规律**：粉丝分布、留存率/分享率/收藏率、关注增长趋势
- **营销趋势**：按月度/季度/年度自动生成数据报告，全面分析不同渠道、不同游轮、不同时期的现状与发展趋势（含客流总体情况、游客画像、负载率、订单效果）
- **分析应用**：游客总体满意度、评价数量及变化趋势、评价云图、各维度评价分析、OTA 平台评价数据占比、游客接待量、性别/年龄/游船时长/购买渠道/团散比/客源地分析

## 8.6 物资数据分析

- 建立物资消耗与单客耗量、餐标达成率等分析机制
- 在月度分析中输出重点单品物资消耗分析报告、餐料航次消耗分析报告
- 精准识别异常消耗，评估消耗合理性
- 建立采购价格数据库：按品类、规格、供应商、采购时间梳理近 3 年主要餐料采购价格和各供应商报价明细
- 分析市场价格波动趋势，建立结构化数据库，为采购谈判、限价调整提供基线支撑

## 8.7 二消产品监控

- 实时监控各二消产品客单价、销量、单客成本、项目收入、转化率
- 基于航次统计数据，分析游客消费口味、时段、偏好、消费关联
- 实时显示库存预警、异常销售等指标
- 形成可视化数据界面，实现客群画像，助力精准营销

# 第九章 用户数据平台与精准营销

> 本章内容来自产品中心数智化项目需求文档。虽不直接描述游轮运营，但用户数据中台的能力是游轮精准营销、会员服务、客源分析的基础设施。

## 9.1 统一用户身份识别与账户体系

- 在一体化平台建立统一的用户账户中心
- 整合来自小程序、官网、OTA、船上消费等渠道的用户身份
- 支持手机号、微信、支付宝等多身份绑定
- 实现用户跨渠道识别与统一管理

## 9.2 用户标签与分群管理

- 支持运营人员为用户自定义标签（如"亲子家庭""高端客户""三峡航线偏好"）
- 按多维度精细化分层筛选：
  - 基础属性：年龄、性别、地域、注册渠道
  - 消费特征：房型、消费频次、产品/航线偏好、二销消费情况
  - 行为特征：浏览轨迹、预订习惯、出行时间
  - 会员等级
- 针对不同分层客群，可定向推送差异化营销内容与优惠政策，实现"千人千策"精准触达

## 9.3 360° 用户画像构建

基于标签采集数据构建全维度用户画像：
- 基础属性：年龄、地域、注册渠道
- 消费特征：客单价、消费频次、偏好产品类型
- 行为特征：活跃时段、浏览深度、转化路径
- 服务偏好：是否使用自助值船、是否参与船上活动
- 支持画像的可视化展示与导出

## 9.4 全渠道用户行为数据采集

在小程序、官网、H5 页面等前端触点部署埋点，采集：
- 浏览行为：页面访问、产品浏览、搜索关键词
- 交互行为：点击、收藏、分享、咨询
- 交易行为：下单、支付、退改
- 服务行为：值船、消费、评价

同步船上消费、客服对话等后端数据至统一数据平台。

## 9.5 行为路径与转化分析

- 支持分析用户从浏览到下单的转化路径，识别关键流失节点
- 支持对比不同渠道（小程序、OTA、分销商）用户的转化效率
- 支持分析产品组合购买行为（如购买船票后是否加购升舱套餐）

## 9.6 多渠道消息触达能力

- 构建统一的消息中心，支持：小程序订阅消息/服务通知、短信、微信公众号模板消息
- 支持根据用户偏好、行为触发自动消息（下单提醒、登船提醒、行程变化通知、活动通知、离船回访）
- 支持运营人员手动创建群发消息，可按用户分群定向推送

## 9.7 营销活动全流程数据追踪

- 所有自营营销活动均实现全维度数据监控与效果分析
- 实时查看：活动参与人数、优惠券发放/领取/使用量、活动带动订单量/交易额、转化率、投入产出比
- 活动结束后自动生成效果分析报告
- 支持多活动效果对比

## 9.8 产品评价分析报表

- 针对当年重新优化的常规产品、新推出的主题产品、新船产品
- 分类统计运行一段时间后与往年同期对比分析
- 数据包括：接待量、来自不同分销渠道的游客满意度数据（具有产品差异化的要素评价）
- 客户满意度与小程序的满意度调查打通数据链接
