## 6.游轮管理
搜索筛选建议：

+ 关键词（游轮名称）

表单字段建议（分布填写，分组展示）：
  
  交互逻辑
1. **顶部步骤条**：清晰展示 1、2、3步的进度状态（当前步高亮，未达步置灰）。
2. **底部操作区**：固定在表单底部，包含"取消"、"上一步"、"下一步"（最后一步变为"提交/生成"）。
3. **按需渲染**：主体区域只渲染当前 Step 对应的表单内容，不要把所有字段堆在一起。

step1基本信息：
**基础标识信息**：游轮名称*（单行文本框）、英文名称*（单行文本框）、游轮代码*（搜索下拉框）、实名制ID*（单行文本框）、船级*（下拉单选）、星级*（下拉单选）
**物理规格信息**：核载客数*（数字输入框）、层数*（数字输入框）、长度（数字输入框）、宽度（数字输入框）、型深（数字输入框）、静水航速（数字输入框）
**设施与工程信息**：电压（数字输入框）、空调系统（单行文本框）、出厂日期*（日期选择器）、末次装修（日期选择器）、首航日期（日期选择器）、装修内容（多行文本区）
**联系人信息**：联系人（单行文本框）、联系人电话（单行文本框）

step2甲板信息：根据step1填写的层数生成block数。每个block包含字段：物理层数（纯文本，X层，X自增生成，最大值等于层数，跨行）、甲板名称*（Input，跨行）、英文名称 *（Input，跨行）、面积（正数Input，跨行）、图片（图片上传）、备注（Input）。甲板设施（单选下拉，不跨行）、营业时间（Select，不跨行）、启用状态（Switch开关）、操作（删除，删除不跨行的设施）、新增设施（按钮，点击后生成一行不跨行的空白设施）

step3：嵌套 rowSpan 表格，甲板级字段（层数、名称）跨设施行合并，rowSpan 等于甲板层数；房型级字段每行独立。房型级字段：房型名称 *（文本框）、英文名称 *（文本框）、图片 *（图片上传）、房型数量 *（数字输入框）、床位数 *（数字输入框）、可加床（数字输入框）、客容量 *（数字输入框）、面积（㎡） *（数字输入框）、阳台（㎡）（数字输入框）、溢价差（数字输入框）、楼层费（数字输入框）、高（m）（数字输入框）、简介（多行文本框）、排序（数字输入框）、按间售卖 *（开关）、游览计划合并展示（开关）。操作：删除、新增设施

表格列建议：序号、游轮代码、游轮名称、游轮系列、船级、星级、载客量、房型数量、层数、长度、型宽、修改人、修改时间

操作：新增游轮、编辑游轮、查看详情（侧边抽屉展示，宽度 ≥ 640px）、删除（二次确认）

## 8.房间管理（含批量生成引擎）
*(注：有别于"舱房类型管理"，此模块针对具体的物理房间实例。需要重点实现批量生成逻辑)*

搜索筛选建议：

+ 房型名称（下拉多选，关联已有的舱房类型）
+ 所在甲板（下拉单选）
+ 房间状态（下拉单选：可用、维修中、锁房）
+ 空间属性（下拉单选：船艏、船中、船艉）
+ 所在游轮（下拉单选）

表单字段建议（单条新增/编辑）：

**基本信息：** 房间号*（文本输入框，唯一校验，如"8001"）、所属游轮*（下拉单选）所属房型*（下拉单选，关联舱房类型）、所在甲板*（下拉单选）、空间位置（单选框：船艏、船中、船艉）。
**房间属性：** 连通房（开关，开启后显示关联房间号的搜索下拉框）、无障碍房间（开关）、视线遮挡（开关，开启后必填遮挡说明文本框）。
**状态与维护：** 当前状态*（单选框：可用、维修中、锁房）、维护备注（多行文本域）。

### 核心功能交互：批量生成房间（独立弹窗或抽屉，宽度 ≥ 800px）

采用 **"分步向导 (Wizard)"** 设计，包含"配置规则"与"数据预览"两步。

**Step 1: 配置生成规则**
+ **基础属性**：所属房型*（下拉单选）、所在甲板*（下拉单选）、空间位置默认值（单选：船艏、船中、船艉，生成后可改）。
+ **命名规则**：房间号前缀（文本框，如输入甲板号 "8"）、起始序号*（数字输入框，如 "01"）、结束序号*（数字输入框，如 "50"）、序号补零（开关，默认开启，例如 1 补齐为 01）
+ **生成策略**：编号类型（单选框：全部连号、仅单号(常用于左舷)、仅双号(常用于右舷)）、排除号码（标签输入框/多项选择，用于跳过忌讳数字，如支持输入 "4", "13", "14" 等，生成时自动跳过包含这些数字的尾号）

**Step 2: 结果预览与微调**
+ 点击"预览生成"后，在下方渲染一个可编辑表格（基于 Step 1 规则在前端内存中生成的房间列表）。
+ 预览表格列：序号、房间号（可直接在表格内修改）、所属房型、甲板、空间位置（支持表格内下拉修改，方便为部分特殊房间标记船艏/船艉）、操作（移除此行）。
+ **交互要求**：允许用户在提交到数据库前，在预览表格中手动删减不需要的房间，或微调个别房间的属性。确认无误后点击"确定生成"才提交接口。

+ 无独立搜索页，码头数据在港口列表中通过树形表格展示

+ 港口新增/编辑弹窗底部增加「码头管理」区域

详情展示：
+ 港口详情抽屉新增「码头列表」卡片，展示每个码头的名称/英文/排序

操作：
+ 与港口共用新增/编辑/删除，无独立 CRUD

### 10.3航次库存看板
入口：航次列表每行操作列「库存」按钮，点击弹出页面。独立页面。
页面结构：左侧月历（月历较大），右侧表格，显示航次对应产品名称。

布局（月历）：
+ 月历控件：展示启航日期所在月份的完整月历（7列×6行网格）
+ 顶部月份导航：左右箭头切换月份
+ 航次日期绿色高亮，今天蓝色标记，选中日期反白
+ 非当月格子留空，非航次日期不显示
+ 点击航次日期可选中

布局（库存表格）：
+ 表格列：舱房类型列表
+ 表格行：物理容量（置灰不可编辑）、总可售（置灰不可编辑）、已售、剩余可售（可编辑）、锁定（可编辑）、应急库存（释放按钮，点击后为0，增加到可售）

批量设置弹窗字段：生效日期*（日期段选择器）、生效星期*（复选框，周一-周日）、表格行（该航段对应产品舱房类型）、表格列字段（物理容量置灰、总可售可编辑、已售、锁定可编辑、应急库存可编辑）、保存按钮

### 10.4航次价格日历
入口：航次列表每行操作列「价格」按钮，点击弹出价格日历页面。独立页面。

数据结构：每条记录 = 航次 + 舱房类型 + 日期，包含四个价格字段（基准价、成人价、儿童价、婴儿价）。

页面布局（左右下布局）：
「月历」组件：居左。按月份维度展示日历，含月份筛选器。开航日期高亮，未开航日期不高亮。点击日期后刷新右侧价格表格。
「表格」组件：居右。横向列：成人票、儿童票、婴儿票等票种。纵向行：该航线下产品关联的房型类型。表格上方有航段切换器（Tab或下拉）。切换后同步刷新价格。价格以已配置好的数据初始化。表格下方有保存和提交按钮。
「价格日志」：表格字段含操作对象、操作时间、操作人。

批量设置弹窗字段：生效日期*（日期段选择器）、生效星期*（复选框，周一-周日）、生效航段*（单选下拉框）、表格（横向票种、纵向房型类型）。
